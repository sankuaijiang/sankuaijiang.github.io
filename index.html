c语言笔记

第一个c语言程序

  1. 程序的第一行 #include <stdio.h> 是预处理器指令，告诉 C 编译器在实际编译之前要包含 stdio.h 文件。
  2. 下一行 int main() 是主函数，程序从这里开始执行。
  3. 下一行 printf(…) 是 C 中另一个可用的函数，会在屏幕上显示消息 “Hello, World!”。 // 是注释 会被编译器忽略
  4. 下一行 return 0; 终止 main() 函数，并返回值 0。

日常使用函数

puts() 打印变量全部字符

gets() 读取用户输入的全部字符串

scanf() 读取用户从键盘输入的数据，并把数据传递给程序

printf() 读取程序中的数据，并把数据显示在屏幕上。

getchar() 读取下一个输入字符，因此程序会等待用户输入，按下Enter（或Return）键（发送一个换行符）。

学习运算符

设int A=10 ；int B=20；

运算符 描述 实例
+ 把两个操作数相加 A + B 将得到 30
- 从第一个操作数中减去第二个操作数 A - B 将得到 -10
* 把两个操作数相乘 A * B 将得到 200
/ 分子除以分母 B / A 将得到 2
% 取模运算符，整除后的余数 B % A 将得到 0
++ 自增运算符，整数值增加 1 A++ 将得到 11
– 自减运算符，整数值减少 1 A– 将得到 9

学习switch-case体系

这个程序是输入一个数字1-7就能告诉你今天是星期几，如果我们许如1，就能得到Monday。输入2就能得到Tuesday。但是输入6却可以得到Saturday和Sunday。是因为我们在case 6中没有加break，switch-case 判断退出是读取到break或者default才会退出，不然会一直读取下去。

循环类型 描述
while 循环 当给定条件为真时，重复语句或语句组。它会在执行循环主体之前测试条件。
for 循环 多次执行一个语句序列，简化管理循环变量的代码。
do…while 循环 除了它是在循环主体结尾测试条件外，其他与 while 语句类似。

二进制转八进制

10001111

每3位为一组用一位八进制数的数字表示，不足3位的要用“0”补足3位，就得到一个八进制数。

010 000 111

2 1 7

二进制转十六进制

10001111

每4位为一组用一位十六进制数的数字表示，不足4位的要用“0”补足4位，就得到一个十六进制数。

1000 1111

起名标识符

  • 标识符只能以英文大小写字母、和下划线(_)开头,而不能用其他任何字符或数字开头。

github环境配置

账号注册

  • 然后就登录邮箱查看 验证码 输入进去
  • 选择free就好了

创建仓库(repository)

owner 不用改 然后Repository 就该你owner名字 +github.io

例如我创建的名字叫SafeMoon1025

Repository name = SafeMoon1025.github.io

  • 选public
  • 勾选Add a README file
  • 然后直接Create repository就ok了

生成ssh

  • 在D盘新建文件夹(blog)

  • 点进去 右键 Git Bash Here

  • ssh-keygen -t rsa -C “邮件地址”

  • 然后慢慢按三次回车

  • 返回github网页点右上角的头像 按setting

  • SSH and GPG keys new SSH keys

  • 然后就去C:\User\Adminisrator.ssh 找到id_rsa.pub 用记事本打开

  • Title: blog key: 直接复制粘贴 然后add keys

  • 然后去test

  • ssh -T git@github.com

启动博客

  • 打开blog文件夹

  • 右键Git Bash Here

    • hexo init
    • hexo g
    • hexo s

  • 给config.yml配置文件

    • deploy:
  type:git
  repository:https://github.com/SafeMoon1025/SafeMoon1025.github.io
  branch:main

    • 然后就安装

      • npm install -g hexo

    • hexo -g hexo -d

    • 用户名输入注册的

    • 密码就在github主页

      • setting
        • Developer setting
          • perosonal access token

    • 名字填blog 全部勾上 点Generate token

    • 然后就可以复制粘贴到那个框框密码上就可以了

博客搭建完如何发布

  • 把写好的文章放在source\_posts里面
  • hexo -s 是部署本地服务器
  • hexo -g 是将本地的文章部署到本地服务器里面
  • hexo -d 是将本地的文章部署到github网站服务器里面
